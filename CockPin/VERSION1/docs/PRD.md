## 1. 프로젝트 개요

핀콕(PinCock)은 제주 올레길에서 발견되는 해양 쓰레기를 손쉽게 **신고**하고 **처리 현황**을 추적할 수 있는 모바일 플랫폼입니다. 제주 올레길은 해안과 인접해 있어 해양 쓰레기가 자주 밀려오지만, 이를 즉각 신고하고 처리 현황을 공유할 수 있는 체계가 부족합니다. PinCock은 이용자가 쓰레기를 발견했을 때 사진과 위치를 앱에 간단히 등록하면, 관계 기관이나 봉사자가 해당 정보를 확인하고 **신속히 수거 작업을 진행**할 수 있도록 돕습니다. 이러한 과정을 통해 지역 주민과 관광객 모두가 환경 보호 활동에 쉽게 동참하고, 제주 자연환경의 신속한 정화에 기여하는 것이 목표입니다.

본 문서는 공모전 제출용으로 작성되었으며, PinCock의 최소 기능 제품(MVP)에 초점을 맞춰 핵심 기능과 개발 계획을 정리합니다.

## 2. 유저 플로우

1. **메인 화면 확인:** 사용자가 앱을 실행하면 PinCock의 소개와 이용 방법이 요약된 메인 화면이 나타납니다. 여기에서 서비스 목적과 사용 방법을 이해한 후, 쓰레기 신고 절차를 시작합니다.
2. **쓰레기 신고 등록:** 올레길을 걷다가 쓰레기를 발견한 사용자는 **쓰레기 신고** 화면으로 이동합니다. 쓰레기 사진을 촬영하거나 갤러리에서 선택하여 업로드하고, 발견 위치의 주소를 입력한 뒤 신고를 제출합니다. 제출과 동시에 지도에 해당 위치에 핀(mark)을 표시하며, 신고 내용이 **확인 대기 중** 상태로 등록됩니다.
3. **쓰레기 현황 확인:** 사용자는 **쓰레기 현황** 화면에서 자신이 올린 신고를 포함한 모든 신고 목록을 확인할 수 있습니다. 방금 등록한 신고 건은 '확인 대기 중'으로 표시되며 목록에 추가됩니다. 이후 관리자가 내용을 검증하여 상태를 **확인됨**으로 변경하고, 실제 쓰레기 수거 작업이 완료되면 **수거완료**로 업데이트됩니다. (MVP 단계에서는 관리자 승인은 간략히 시뮬레이션)
4. **상태 모니터링 및 공유:** 사용자는 지속적으로 쓰레기 현황 화면을 통해 신고들의 처리 상태 변화를 모니터링할 수 있습니다. 필요에 따라 **지역 필터**를 사용해 제주시 또는 서귀포시 등 특정 지역의 신고만 추려볼 수 있고, **읍·면·동 단위**로도 세부 조회가 가능합니다. 또한 최근에 수거 완료된 신고들은 별도 표시되어 있어 진행 상황을 한눈에 파악할 수 있습니다. 사용자는 자신의 신고가 처리된 것을 확인하고, 다른 쓰레기 문제에도 관심을 가지며 참여 의식을 높일 수 있습니다.

## 3. 핵심 기능 (MVP)

MVP 단계에서 구현되는 핵심 기능은 다음과 같습니다:

| **화면 / 기능** | **핵심 내용**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **메인 화면**   | 앱 소개 및 이용 가이드 제공:<br/>- PinCock 서비스 소개 (목적 및 중요성 안내)<br/>- 앱의 주요 기능과 사용자 이용 흐름 설명<br/>- 기대 효과 강조 (참여를 통한 해양 환경 개선 등)                                                                                                                                                                                                                                                                                                                                       |
| **쓰레기 신고** | 해양쓰레기 제보 기능:<br/>- **사진 업로드:** 사용자가 촬영한 쓰레기 사진을 로컬 저장소에서 선택하여 첨부<br/>- **위치 입력:** 쓰레기 발견 위치의 주소를 입력하여 위치 정보 등록 (지도 연동 가능)<br/>- **핀 생성:** 입력된 위치를 기반으로 지도에 핀 마커를 생성하여 신고 지점을 시각화                                                                                                                                                                                                                              |
| **쓰레기 현황** | 신고 현황 열람 및 필터링:<br/>- **전체 현황:** 모든 신고 내역을 목록 또는 지도에 표시하고, 각 신고의 상태(확인 대기 중/확인됨/수거완료)를 표시<br/>- **지역별 보기:** 행정구역별 필터 적용 – 제주시 또는 서귀포시로 구분하여 해당 지역의 신고만 보기<br/>- **읍·면·동별 보기:** 세부 행정구역 단위로 필터링 – 선택한 읍/면/동의 신고 내역만 조회 _(제주시: 4읍·3면·19동, 서귀포시: 3읍·2면·12동 데이터 반영)_<br/>- **최근 수거 표시:** 최근에 **수거완료**된 신고 내역을 강조 표시하여 최신 정화 활동을 한눈에 파악 |

## 4. 기술 스택

- **플랫폼:** React Native (JavaScript, HTML/CSS) – 크로스플랫폼 개발을 통해 Android/iOS 및 웹 브라우저에서 모두 동작
- **개발 형태:** 웹앱(Web App) – 모바일 기기 브라우저 또는 웹뷰(WebView)로 접근 가능한 형태로 구현 (네이티브 앱처럼 설치 없이 사용)
- **지도 및 위치 서비스:** 지도 API 및 지오코딩 활용 – 사용자가 입력한 주소를 좌표로 변환해 지도에 핀 표시 (예: 카카오 지도 API 또는 Google Maps API 활용)
- **데이터 저장:** MVP 단계에서는 별도 서버 없이 **로컬 저장소(Local Storage)**를 사용하여 신고 내용과 이미지를 저장 및 관리 (단말기 내 임시 보관). 추후 확장을 위해 백엔드와의 연동을 고려함.

## 5. MVP 이후 추가 개선 사항

- **백엔드 서버 연동:** 클라우드 데이터베이스나 전용 서버 도입으로 신고 데이터를 영구적으로 저장하고, 여러 사용자가 동시에 이용할 수 있도록 지원
- **회원 및 인증 기능:** 사용자 회원가입/로그인을 추가하여 제보자 프로필 관리 및 본인이 등록한 신고 내역을 손쉽게 확인 (신뢰성 있는 제보 환경 구축)
- **관리자 도구 강화:** 관리자가 앱이나 웹을 통해 접수된 신고를 검증하고 상태를 변경(확인됨/수거완료)할 수 있는 관리자 모드 또는 백오피스 시스템 추가
- **알림 기능:** 신고 처리 단계별 푸시 알림 또는 알림 메시지 제공 – 신고 접수 확인, 처리 완료 시 제보자에게 알림을 보내고, 필요시 해당 지역 봉사자들에게 수거 요청 알림
- **UI/UX 고도화:** 지도 상 마커 디자인 개선(상태별 아이콘 색상 구분, 클러스터링 등) 및 전반적인 화면 디자인 개선, 앱 사용 방법에 대한 튜토리얼 제공
- **추가기능 확장:** 사용자 참여 독려를 위한 포인트/뱃지 등 **게이미피케이션** 요소 도입, 다국어 지원을 통한 외국인 올레 방문객 참여 유도, 제주도 외 다른 지역 해안 쓰레기 제보로 범위 확대 등 향후 기능 개선 사항을 단계적으로 검토합니다.
